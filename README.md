**GitHub Version Control Implementation using Branching and Pull Requests**
**Project Description**

This project demonstrates how to use Git and GitHub for version control in a team environment. It includes creating a repository, working with multiple branches, making changes, and merging them using pull requests while handling version control workflow.

**Tools Used**

Git
GitHub
Git Bash / Terminal

**Branch Structure**

main → Stable production code
feature-login → Login module
feature-dashboard → Dashboard module
feature-api → API development

**Workflow Followed**

Created a GitHub repository
Cloned repository to local system
Created separate branches for features
Made changes in each branch
Committed changes using Git
Pushed branches to GitHub
Created Pull Requests
Performed code review
Merged branches into main
Deleted feature branches after merging
